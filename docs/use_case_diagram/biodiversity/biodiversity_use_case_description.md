
本用例图描述了**生物多样性监测子系统**中，不同角色围绕“监测数据”的上传、复查、审核和统计分析等主要功能。

---

## 一、系统中有哪些角色？

系统中主要涉及两类用户，以及系统本身的自动处理逻辑：

1. **生态监测员（Ecological Monitor）**  
    负责采集并上传监测数据，并对系统标记的数据进行复查。
    
2. **数据分析师（Data Analyst）**  
    负责对复查后的监测数据进行最终审核，并生成统计分析结果。
    
3. **系统（System）**  
    自动完成监测数据的完整性校验，并设置数据初始状态。
    

---

## 二、生态监测员可以做什么？

生态监测员主要完成以下三个用例：

1. **Upload Monitoring Record（上传监测记录）**
    
    - 生态监测员将现场采集到的物种监测数据上传到系统。
        
    - 系统会自动对数据完整性进行校验。
        
2. **Recheck Monitoring Record（复查监测记录）**
    
    - 当系统将数据标记为“待核实”后，生态监测员对这些记录进行业务复查。
        
    - 复查完成后，数据状态被更新为“已复查（rechecked）”。
        
3. **View Monitoring Statistics Report（查看监测统计报表）**
    
    - 生态监测员可以查看与其负责区域相关的生物多样性监测统计结果。
        

---

## 三、系统自动完成什么？

系统在监测数据上传后，会自动执行以下操作：

- **Verify Monitoring Record Integrity（校验监测记录完整性）**
    
    - 系统检查监测记录是否缺少必要字段。
        
    - 若校验通过，系统将数据状态自动设置为 `pending`（待核实）。
        
    - 该过程由系统自动完成，不需要人工参与。
        

---

## 四、数据分析师可以做什么？

数据分析师主要负责以下用例：

1. **Valid Monitoring Record（审核监测记录）**
    
    - 对已经复查过的监测记录进行最终审核。
        
    - 审核通过后，将数据状态更新为 `valid`（有效数据）。
        
2. **Generate Monitoring Statistics Report（生成监测统计报表）**
    
    - 基于有效的监测数据，生成多维度的统计分析报表。
        
3. **Maintain Analysis Model and Thresholds（维护分析模型与阈值）**
    
    - 维护用于数据分析的模型参数和判断阈值，为统计分析提供支持。
        

---

## 五、用例之间的关系（一句话版）

- **上传监测记录** 一定会 **包含** 系统的自动校验过程。
    
- **复查监测记录** 和 **审核监测记录** 是在上传之后按顺序发生的人工处理步骤。
    
- **查看监测统计报表** 的功能依赖于已经生成的统计结果。
    

---

## 六、整体流程一句话总结

> 生态监测员上传数据 → 系统自动校验并标记为待核实 → 生态监测员复查 → 数据分析师终审 → 生成并查看监测统计报表。
