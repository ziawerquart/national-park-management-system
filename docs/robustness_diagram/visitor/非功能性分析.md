非功能性分析
1. 安全性分析

身份验证：游客入园时需要进行身份验证，确保只有持有效预约的游客能够入园。系统会检查身份证号和游客信息的匹配。

数据加密与安全存储：游客的个人信息和预约记录必须使用加密方式存储，并且采取适当的权限控制，防止数据泄露或非法访问。

流量控制预警：当在园游客超过预警阈值时，系统会自动发出警告，确保在游客密集时采取限流措施。

2. 完整性约束

数据唯一性：游客的身份证号和游客 ID必须唯一，以确保数据的完整性和准确性。

外键约束：预约记录必须关联到一个有效的游客 ID，并且轨迹数据与游客 ID 关联，确保数据之间的关系正确。

时间一致性：入园时间必须早于离园时间，系统需校验时间的合理性。
