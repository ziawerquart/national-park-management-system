# National Park Management System

## 📋 项目概述

国家公园智慧管理与生态保护系统是一个集生物多样性监测、生态环境监测、游客智能管理、执法监管、科研数据支撑于一体的综合平台。本系统通过整合数据库技术、物联网感知和数据分析手段，实现国家公园管理的数字化、智能化闭环，为生态保护与科学管理提供精准决策依据。

## 🎯 项目背景

国家公园作为生态保护的核心载体，面临着生态监测精细化不足、游客管理粗放、执法响应滞后、科研数据分散等管理痛点。传统依赖人工巡查、线下记录的管理模式，已难以满足"严格保护、科学利用"的核心需求。本系统旨在通过现代数据库技术解决这些挑战。

## 🌟 核心业务线

本系统包含5条核心业务线，每条业务线由一名团队成员完整负责：

### 1. 生物多样性监测业务线
- **核心功能**：物种信息管理、监测记录管理、栖息地信息管理
- **数据实体**：物种信息、监测记录、栖息地信息
- **业务场景**：生态监测员采集物种数据，系统自动校验数据完整性，数据分析师审核待核实数据

### 2. 生态环境监测业务线
- **核心功能**：监测指标管理、环境数据采集、设备状态监控
- **数据实体**：监测指标信息、环境监测数据、监测设备信息
- **业务场景**：物联网设备实时采集环境数据，系统自动预警异常数据，技术人员定期校准设备

### 3. 游客智能管理业务线
- **核心功能**：游客预约管理、实时轨迹追踪、区域流量控制
- **数据实体**：游客信息、预约记录、游客轨迹数据、流量控制信息
- **业务场景**：游客线上预约入园，系统实时追踪轨迹，区域流量超阈值时自动预警

### 4. 执法监管业务线
- **核心功能**：非法行为记录、执法调度管理、视频监控管理
- **数据实体**：执法人员信息、非法行为记录、执法调度信息、视频监控点信息
- **业务场景**：智能视频监控识别非法行为，系统自动调度执法人员，跟踪处置全流程

### 5. 科研数据支撑业务线
- **核心功能**：科研项目管理、数据采集记录、成果共享管理
- **数据实体**：科研项目信息、科研数据采集记录、科研成果信息
- **业务场景**：科研人员提交项目申请，录入或调用采集数据，上传管理科研成果

## 🛠️ 技术栈

- **数据库**: MySQL 8.0
- **后端语言**: Python 3.9
- **ORM框架**: SQLAlchemy
- **设计工具**: PowerDesigner (E-R图设计), VS Code (代码开发)
- **版本控制**: Git + GitHub
- **测试框架**: pytest
